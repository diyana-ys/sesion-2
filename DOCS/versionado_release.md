# Versionado y Release

Se sigue [SemVer](https://semver.org/) para versionado.

- Incrementar MAJOR cuando haya cambios incompatibles
- Incrementar MINOR para nuevas funcionalidades compatibles
- Incrementar PATCH para correcciones de bugs

Los tags de Git seguirán la misma convención (`vMAJOR.MINOR.PATCH`).

## Release notes

Cada release debe incluir un resumen de cambios basado en los mensajes de commit.

> Ejemplo:
> - `v1.2.0` - Añade endpoint de búsqueda de actores.
